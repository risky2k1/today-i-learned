## DevOps

Hôm nay mình phát hiện `docker stats` chỉ hiển thị CPU usage tương đối so với tổng CPU của host, do đó con container "chỉ" dùng 5% CPU mà host có 8 core thì thực tế nó đang dùng rất nhiều tài nguyên. Nên kiểm tra luôn `--cpus` và `--memory` khi chạy container production để tránh trường hợp container "ăn" hết tài nguyên hệ thống.

*TIL này giúp mình chủ ý thiết lập resource limits kỹ hơn trong các file docker-compose cho dự án.*
