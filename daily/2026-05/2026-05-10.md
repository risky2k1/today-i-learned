## DevOps

- Terraform `count` vs `for_each`: `count` dùng index-based, dễ bị lỗi ordering khi thêm/xóa resource; `for_each` dùng map/set nên ổn định hơn khi scale.

- Mình sẽ refactor luôn cái state file đang dùng `count` để đỡ bị breaking change mỗi lần thêm environment.


## Programming

- Hôm nay mình học cách dùng `match` expression trong Python 3.10 để thay thế chuỗi các `if‑elif‑else` phức tạp.  
- `match` cho phép pattern matching trực quan, hỗ trợ so sánh giá trị, cấu trúc danh sách và dict.  
- Ví dụ: `match data: case {"type": "user", "id": id}: ...` giúp mã ngắn gọn và dễ bảo trì.  

*Cảm giác vừa khám phá một cú pháp mới, mình sẽ thử đưa `match` vào dự án API hiện tại để giảm bớt logic rối.*
