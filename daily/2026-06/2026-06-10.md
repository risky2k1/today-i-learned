## Automation

Tôi vừa tìm hiểu về **GitHub Actions** để tự động triển khai các dự án Python lên Render, giúp việc cập nhật version thực sự “đơn giản hơn bụi.”  
Anh **workflow YAML** ghi nhận các bước:  
- `actions/checkout` để lấy mã nguồn  
- `pip install -r requirements.txt` để cài thư viện  
- `pytest` chạy test, và `render deploy` gửi qua API.  

Dù vẫn chưa tích hợp `secrets` cho deploy key, nhưng đã làm quen với cách đặt biến môi trường và cấu hình nhỏ nhất!  
**Plan:** Bắt đầu viết một script nhỏ để tự động cập nhật Dockerfile khi có thay đổi trong ```requirements.txt```.


## Programming

Hôm nay TIL: `structuredClone(obj)` clone được object sâu hơn và an toàn hơn `JSON.parse(JSON.stringify(obj))`, nhất là với `Date`, `Map`, `Set` hoặc dữ liệu có `undefined`.  
Tuy nhiên vẫn cần cẩn thận nếu object có circular reference hoặc chứa function.  
Plan: dùng `structuredClone()` trong các case cần deep copy sạch, còn clone response JSON thì `JSON` vẫn đủ nhanh.
