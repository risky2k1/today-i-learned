## DevOps

- Khi cấu hình CI/CD, mình mới nhận ra việc tận dụng `cache` cho các thư viện dependency (như `node_modules` hay `.m2`) giúp giảm thời gian build pipeline từ 5 phút xuống còn chưa đầy 1 phút.
- Việc quản lý secret qua biến môi trường của GitHub Actions an toàn hơn nhiều so với việc hardcode trực tiếp vào file config.

*Note: Phải rà soát lại toàn bộ các repo cũ để tối ưu hóa cache, đỡ tốn tài nguyên runner.*
