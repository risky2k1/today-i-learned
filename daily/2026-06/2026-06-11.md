## Programming

- Gần đây mình phát hiện ra sự khác biệt giữa `||` và `??` khi xử lý giá trị mặc định: `||` sẽ trả về giá trị thứ hai nếu giá trị đầu là falsy (0, "", false, null, undefined), còn `??` chỉ trả về giá trị thứ hai khi gặp `null` hoặc `undefined`.
- Ví dụ: `0 || "default"` trả về `"default"`, nhưng `0 ?? "default"` trả về `0`.
- Điều này đặc biệt hữu ích khi xử lý dữ liệu từ API hoặc form input mà không muốn làm mất giá trị falsy hợp lệ như số 0 hoặc chuỗi rỗng.

Mình vừa refactor một số hàm xử lý dữ liệu cũ trong dự án, dùng `??` thay vì `||` - code ngắn gọn và đúng ngữ nghĩa hơn rất nhiều!
