## Automation

- Khi sử dụng GitHub Actions, mình phát hiện việc khai báo `cache` cho npm dependencies giúp thời gian CI giảm trung bình 40% so với mỗi lần cài mới.  
- Thêm bước `restore-cache` và `save-cache` trong workflow, kèm `key: ${{ runner.os }}-node-${{ hashFiles('package-lock.json') }}` để cache được cập nhật khi lock file thay đổi.  

**Nhận xét:** Việc tối ưu cache không chỉ tiết kiệm thời gian mà còn giảm tải tài nguyên runner, mình sẽ áp dụng cùng cho các dự án Java và Python trong tuần tới.

##[Programming]
Hôm nay mới biết pattern **"Parse, don't validate"** khi xử lý input: thay vì viết hàm `validate(input) -> bool` rồi ép team nhớ check trước khi dùng, ta viết hàm `parse(input) -> Result<ValidType, Error>`. Code sau đó chỉ làm việc với `ValidType` (đã đảm bảo sạch) nên loại bỏ hẳn các `if` check rác rưởi và null pointer exception ở business logic.

> *Áp dụng ngay cho module nhập liệu CSV hôm nay, thấy code đỡ "nhức nách" hẳn.*

## [Programming]

Hôm nay biết thêm pattern **Null Object Pattern** thay vì check `if (obj != null)` khắp nơi: tạo một class implement interface đó nhưng các method để rỗng/return default value (ví dụ `EmptyUser` implements `User`). Code sạch hơn, tránh NullPointerException và giảm cognitive load khi đọc logic nghiệp vụ.

> *Sẽ áp dụng thử cho module billing đang full `if (user == null) return defaultFee;` xem sao.*
