## DevOps

- Khi triển khai Docker Compose, dùng `build:` thay vì `image:` giúp tự động rebuild image mỗi lần code thay đổi, tiết kiệm thời gian khi phát triển local.
- Thêm `depends_on:` với `condition: service_healthy` (Docker Compose v3.9+) để tránh MongoDB bắt đầu trước khi database sẵn sàng – giúp tránh lỗi nhanh chóng.
- Đánh giá: Sử dụng `--quiet-pull` khi pull images để giảm output, giữ log gọn gàng hơn.

**Lời nhận xét**: Mình có dự định thử `docker compose up --build` trong pipeline GitHub Actions để tự động kiểm tra tính tương thích Build mỗi khi push.
