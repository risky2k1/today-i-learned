## Programming

Hôm nay tìm hiểu về `git bisect` để debug nhanh khi gặp bug được giới thiệu trong commit nào đó. Lệnh này tự động binary search qua lịch sử commit để tìm commit "bad" đầu tiên. Rất hữu ích khi dự án có nhiều commit và cần pinpoint nguyên nhân.

- Dùng `git bisect start` để bắt đầu, `git bisect bad` với commit hiện tại (có bug), `git bisect good <commit>` với commit cũ (không có bug).
- Git sẽ tự động checkout các commit ở giữa và hỏi bạn từng cái có "good" hay "bad".
- Kết thúc bằng `git bisect reset` để quay về trạng thái ban đầu.

Plan: Lần sau gặp bug regression sẽ thử ngay để tiết kiệm thời gian debug.
