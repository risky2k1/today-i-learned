## DevOps

Học được cách dùng **multi-stage builds trong Docker** để tối ưu size image. Thay vì build và chạy trong cùng 1 stage, ta có thể tách riêng: stage 1 build source code với full dependencies, stage 2 copy artifacts cần thiết vào image runtime tối giản.

Ví dụ với Node.js:
```dockerfile
FROM node:18 AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

FROM node:18-alpine AS runtime
COPY --from=builder /app/dist ./dist
COPY --from=builder /app/node_modules ./node_modules
CMD ["node", "dist/index.js"]
```
Giảm được ~60% size image so với cách cũ.

Hôm nay tìm hiểu về Kubernetes Rolling Update strategy. Khi deploy phiên bản mới, Kubernetes sẽ tạo pods mới song song với pods cũ, sau đó dần dần tăng tỉ lệ pods mới và giảm pods cũ cho đến khi hoàn tất. Điều này giúp giảm downtime và đảm bảo tính availability của hệ thống.
