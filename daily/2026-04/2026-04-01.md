## DevOps

Hôm nay tìm hiểu về **Docker Compose với multiple environments**. Có thể dùng `.env` file và `docker-compose.override.yml` để tách biệt config dev/prod. Điều này giúp tránh hardcode biến môi trường vào file chính.

- `.env` chứa các biến dùng chung.
- `docker-compose.override.yml` ghi đè cho dev.
- Dùng `docker-compose -f docker-compose.yml -f docker-compose.prod.yml up` để chạy production.

Cảm thấy cách này sạch và dễ maintain hơn việc clone file rồi sửa tay. Sắp tới sẽ áp dụng cho dự án cá nhân để test CI/CD.
