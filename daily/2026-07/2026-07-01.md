## [Programming]

Hôm nay biết thêm pattern **Null Object** thay vì check `if (obj != null)` lặp đi lặp lại khắp codebase. Thay vì return `null`, ta return một object implements interface đó nhưng các method để rỗng hoặc trả giá trị default an toàn (ví dụ `EmptyUser`, `NullLogger`). Code client gọi method bình thường mà không lo `NullPointerException`, logic chính sạch sẽ hơn hẳn.

> 💡 *Sẽ áp dụng thử cho module payment gateway đang维护, chỗ nào hay throw NPE vì config missing thì đổi sang `NullPaymentProvider` cho đỡ phải try-catch vùi dập.*



## Programming  
- Hôm nay mình khám phá cách dùng context manager (`with` statement) trong Python để tự động quản lý tài nguyên (file, kết nối DB).  
- Giúp tránh lỗi "memory leak" hoặc file bị khóa khi quên `close()` do exception xảy ra.  
- Ví dụ: `with open('file.txt', 'r') as f:` giúp đảm bảo file luôn được giải phóng sau khi xử lý.  

*Mình sẽ áp dụng ngay vào dự án hiện tại để code sạch hơn và giảm bug do tài nguyên chưa được giải phóng.*
